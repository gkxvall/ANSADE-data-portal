# UI Inspiration Guide for the `Inso` Folder

## Purpose

The project root will contain an `Inso` folder with screenshots and images that Codex must study before implementing the interface.

The images are references for visual direction, not templates to copy exactly.

## Required Codex Workflow

Before implementing or redesigning a page, Codex must:

1. Inspect every image in `Inso`.
2. Identify repeated visual patterns.
3. Record the useful patterns in the stage report.
4. Map those patterns to the ANSADE data hierarchy and workflows.
5. Build a coherent design system rather than reproducing screenshots independently.

## Elements to Analyze

Codex should study:

- page composition
- navigation style
- sidebar behavior
- header structure
- card proportions
- typography hierarchy
- spacing rhythm
- border radius
- shadows
- table styling
- chart containers
- filters
- search controls
- tabs
- empty states
- loading states
- color usage
- responsive behavior

## Design Priorities

The final design should feel:

- modern
- institutional
- trustworthy
- calm
- data-focused
- accessible
- spacious
- responsive

It must not feel like an admin template with random widgets.

## ANSADE-Specific Adaptation

The inspiration must be adapted to:

- categories
- themes
- datasets
- statistical metadata
- tables
- filters
- charts
- comparisons
- exports
- data quality

The design should make the hierarchy obvious without forcing users through excessive navigation.

## Design-System Output

Codex should define:

- primary and secondary colors
- neutral palette
- semantic colors
- typography scale
- spacing scale
- radius scale
- shadow scale
- container widths
- responsive breakpoints
- card variants
- button variants
- form controls
- table variants
- chart-card conventions

## Rules

- Do not copy logos, brands, names, or copyrighted text from inspiration images.
- Do not reproduce a screenshot pixel-for-pixel.
- Do not add decorative complexity that harms statistical readability.
- Do not use images where real data visualization is more appropriate.
- Do not invent ANSADE content.
- Use real imported data when available.
- Keep charts readable and restrained.
- Support both French and Arabic layout requirements later, even if the first implementation is French-first.

## Stage Reports

For every UI-heavy stage, the report must explain:

- which inspiration patterns were used
- where they were adapted
- why they fit the ANSADE portal
- which patterns were rejected and why
